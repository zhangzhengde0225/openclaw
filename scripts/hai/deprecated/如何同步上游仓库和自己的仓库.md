你 fork 了一个公开项目（比如原始项目是 `upstream_repo/openclaw`），现在你的 GitHub 仓库是：

```
origin: https://github.com/zhangzhengde0225/openclaw.git
```

你想**同步上游（原项目）的最新更改到你自己的 fork 中**，这是非常常见的操作。下面是标准做法：

---

### ✅ 步骤 1：添加上游（upstream）远程仓库

在你的本地 Git 仓库中，运行以下命令（只需做一次）：

```bash
git remote add upstream https://github.com/ORIGINAL_OWNER/openclaw.git
```

> 🔔 请将 `ORIGINAL_OWNER` 替换为**原始项目作者的 GitHub 用户名**（不是你自己的）。  
> 例如，如果原项目是 `https://github.com/johndoe/openclaw`，那就写：
> ```bash
> git remote add upstream https://github.com/johndoe/openclaw.git
> ```

你可以通过访问你 fork 的 GitHub 页面顶部看到 “forked from johndoe/openclaw” 来找到原作者。

验证是否添加成功：
```bash
git remote -v
```
你应该看到：
```
origin    https://github.com/zhangzhengde0225/openclaw.git (fetch)
origin    https://github.com/zhangzhengde0225/openclaw.git (push)
upstream  https://github.com/ORIGINAL_OWNER/openclaw.git (fetch)
upstream  https://github.com/ORIGINAL_OWNER/openclaw.git (push)
```

---

### ✅ 步骤 2：拉取上游最新代码

```bash
# 获取 upstream 的所有更新（但不合并）
git fetch upstream
```

---

### ✅ 步骤 3：切换到你的主分支（通常是 main 或 master）

```bash
git checkout main
# 或者
git checkout master
```

> 💡 不确定用哪个？运行 `git branch` 看当前默认分支，或看 GitHub 上你仓库的默认分支名。

---

### ✅ 步骤 4：将上游的更改合并到你的分支

```bash
git merge upstream/main
```
> 如果上游用的是 `master`，就写 `upstream/master`。

这会把原项目的最新提交合并到你的本地分支。

---

### ✅ 步骤 5：推送到你自己的 GitHub（origin）

```bash
git push origin main
```

这样，你的 fork 就和原项目同步了！

---

### 🔄 后续同步

以后每次想同步，只需重复：

```bash
git fetch upstream
git checkout main
git merge upstream/main
git push origin main
```

或者更简洁地（如果你在 main 分支）：
```bash
git pull upstream main
git push origin main
```

> ⚠️ 注意：`git pull upstream main` = `fetch + merge`，效果一样。

### ❓常见问题

#### Q：我改过一些文件，合并时冲突了怎么办？
A：Git 会提示冲突文件。你需要手动编辑解决冲突，然后：
```bash
git add .
git commit
git push origin main
```

#### Q：能不能自动同步，不用手动操作？
A：GitHub **不会自动同步 fork**。但你可以用 GitHub Actions 写一个 workflow 自动拉取并推送（较复杂），或者使用第三方工具（如 [https://github.com/sponsors/wei](https://github.com/wei) 的 `repo-sync` action）。

### 🔒 补充建议（绕过你之前的代理 SSL 问题）

如果你之前因为代理导致 `git pull` 失败，但在同步 `upstream` 时也遇到同样问题，可以：

- 暂时关闭代理（如果网络允许）：
  ```bash
  unset http_proxy https_proxy
  git fetch upstream
  ```
- 或者配置 Git 使用 SSH（更稳定）：
  ```bash
  git remote set-url upstream git@github.com:ORIGINAL_OWNER/openclaw.git
  ```
